1. Subir e Criar o Ambiente
Estes são os comandos para colocar tudo no ar:

docker-compose -f docker-compose.overview.yml up -d: Sobe todos os serviços em segundo plano (detached mode). É o que você mais usará para liberar o terminal.

docker-compose -f docker-compose.overview.yml up -d --build: Força o Docker a reconstruir as imagens dos seus serviços (User, Product, etc.) antes de subir. Use sempre que alterar o código Java ou o Dockerfile.

docker-compose -f docker-compose.overview.yml up --no-deps <serviço>: Sobe apenas um serviço específico sem tentar subir as dependências dele.

2. Parar e Remover
Comandos para "limpar" o que está rodando:

docker-compose -f docker-compose.overview.yml stop: Apenas pausa os containers. Eles continuam existindo e mantêm o estado.

docker-compose -f docker-compose.overview.yml down: O comando "limpa-tudo". Para os containers, remove as redes e apaga os containers. Importante: Seus dados no Postgres não somem porque você usou volumes.

docker-compose -f docker-compose.overview.yml down -v: Remove tudo, inclusive os volumes (cuidado: isso apaga os dados do banco de dados).

3. Monitoramento e Debug (Essencial para o Insomnia)
Se o link no Insomnia não funcionar, use estes comandos:

docker ps: Lista todos os containers rodando, o ID deles e, principalmente, as portas mapeadas (ex: 8081->8080).

docker-compose -f docker-compose.overview.yml logs -f: Mostra os logs de todos os microsserviços em tempo real. Ótimo para ver o Spring Boot iniciando.

docker-compose -f docker-compose.overview.yml logs -f <serviço>: Mostra logs de apenas um serviço (ex: product-service).

docker-compose -f docker-compose.overview.yml exec <serviço> bash: Permite que você "entre" dentro do container para verificar arquivos (muito útil para ver se o app.jar foi copiado certo).

4. Manutenção de Imagens
docker images: Lista todas as imagens baixadas (como a do Eclipse Temurin ou Postgres).

docker image prune: Remove imagens inúteis que ficam "soltas" após vários builds, liberando espaço no seu HD.



# derrubar as imagens e subi-las de novo 
docker-compose -f docker-compose.overview.yml down -v
docker system prune -f
docker-compose -f docker-compose.overview.yml build --no-cache
docker-compose -f docker-compose.overview.yml up -d


# remover algum container orfão
docker-compose -f docker-compose.overview.yml down -v --remove-orphans
docker system prune -f -a
docker-compose -f docker-compose.overview.yml up -d
